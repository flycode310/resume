# Agent 开发工程师简历撰写

专门用于撰写应聘 **Agent 开发工程师 / LLM Engineer / AI Application Engineer** 岗位的简历。

## 核心原则

1. **岗位强相关优先**：把与 Agent / LLM 强相关的经历前置,弱相关的经历压缩或省略。
2. **量化业务价值**：每个项目必须有可量化的指标(准确率、调用成本、QPS、用户数、节省工时等),拒绝"负责""参与""协助"这类含糊动词。
3. **技术深度可验证**：写出来的技术点要能在面试中讲出原理、踩坑、权衡,不堆砌名词。
4. **STAR 结构**:每个项目用 Situation(背景) → Task(目标) → Action(技术方案 + 个人贡献) → Result(成果) 组织。
5. **关键词命中**:针对 JD 中的技术栈词汇(LangGraph、MCP、RAG、Function Calling 等)做关键词匹配,过 HR/ATS 筛选。

## 工作流程

### 阶段 1:信息收集

在动笔前,通过 `AskUserQuestion` 工具向用户收集以下信息(可一次性问完,也可分批询问):

**必问项**:
- 基本信息(姓名、年限、学历、目标公司类型:大厂 / 初创 / 外企)
- 目标岗位 JD(粘贴或描述),用于关键词匹配
- 现有简历(如有,Read 进来作为基础迭代)
- 最近 2-3 个最能体现 Agent 能力的项目(项目背景、个人角色、技术栈、成果数字)
- 是否有开源项目 / 技术博客 / GitHub(强加分项)

**选问项**:
- 期望薪资、地点
- 是否需要英文版

### 阶段 2:技术栈梳理

根据 `references/agent-engineer-keywords.md`,与用户对齐技术栈描述:
- 哪些是**精通**(能讲原理、改源码、生产落地过)
- 哪些是**熟悉**(用过、了解原理)
- 哪些是**了解**(看过文档、写过 demo)

**禁止虚标**——精通级别的技术,面试官一定会深挖。

### 阶段 3:简历生成

使用 `templates/resume-template.md` 作为骨架,按以下顺序填充:

1. **个人信息** - 一行搞定,不要照片、不要籍贯、不要婚姻状况
2. **求职意向** - 一行,明确岗位 + 城市
3. **技术栈** - 分级标注(精通/熟悉/了解),按 Agent 相关度排序
4. **项目经验** - 3 个为佳,最相关的放最前面,每个项目 5-8 行
5. **工作经历** - 公司 + 时间 + 职位,核心成果 2-3 条
6. **教育背景** - 一行,GPA/排名只在 > 前 20% 时写
7. **其他加分项** - 开源、博客、专利、竞赛、社区分享

### 阶段 4:针对性优化

生成后用 `references/review-checklist.md` 自查,重点检查:
- 是否每个项目都有数字
- 是否避免了禁用词(详见 checklist)
- 关键词覆盖率(对照 JD)
- 一页能否打完(中文简历建议 1 页,3 年以上经验可 2 页)

### 阶段 5:产出文件

- 主文件:`resume_{姓名}_{目标岗位}.md`,写到用户当前工作目录
- 如用户要 PDF,提示使用 pandoc / Typora / 在线工具转换
- 如用户要英文版,翻译并调整文化差异(英文简历更强调 ownership 和 impact)

## 常见陷阱

1. **堆砌名词不讲深度**:写了 "LangGraph、AutoGen、CrewAI、MetaGPT" 但每个都只是 hello world——面试官一问就露馅。**只写你能讲清楚原理和踩坑的**。

2. **照抄 JD**:JD 写了什么就在简历技能里加什么,但项目里完全没体现——HR 也许过,技术面 30 秒崩盘。

3. **不写数字**:"提升了系统效率"是废话。"通过引入语义缓存,把 LLM 调用成本从 $0.8/次 降到 $0.15/次,月度节省 $4200" 才是简历。

4. **个人贡献不清晰**:团队项目里要明确"我负责什么、我设计了什么、我解决了什么问题",不要写成项目介绍。

5. **过期技术栈优先级过高**:把 2022 年的 LangChain 0.0.x 当主要卖点,而不写 2025-2026 的 MCP / Agent SDK / Computer Use 等新栈——会被认为没跟进。

## 示例对话开头

当用户说"帮我写简历"时,你应该:

1. 调用 `AskUserQuestion` 一次性收集核心信息(姓名、年限、目标 JD、最近项目)
2. 在用户提供信息后,Read 现有简历(如有)
3. 调用 `Read references/agent-engineer-keywords.md` 准备技术栈对齐
4. 按"阶段 2-5"流程产出

不要一上来就开始写——**先问清楚,再动笔**。
