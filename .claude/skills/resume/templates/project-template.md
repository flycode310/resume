## 检查清单

填完后逐项核对:

- [ ] 项目背景有**量化痛点**(数字)
- [ ] 技术方案 4-6 条,**每条有技术名 + 效果数字**
- [ ] 个人贡献 3-4 条,**明确个人 ownership**
- [ ] 项目成果至少 3 条,**业务数字 + 技术数字**
- [ ] 没有出现禁用词("负责"、"参与"、"协助"、"配合")
- [ ] 技术关键词覆盖目标 JD 的高频词

## 反面例子(不要这样写)

❌ **背景描述空洞**
> "为了提升公司效率,我们决定引入 AI Agent 技术。"
✅ 改为:"客服日均 8000 工单,60% 重复,人力成本 ¥40W/月,需要自动化解决方案。"

❌ **技术堆砌不讲深度**
> "使用了 LangChain、AutoGen、CrewAI、MetaGPT 等多种 Agent 框架。"
✅ 改为:"基于 LangGraph 设计状态机式 Agent,选型对比:LangGraph 比 AutoGen 状态管理更显式,比 CrewAI 更适合 deterministic flow。"

❌ **个人贡献模糊**
> "参与了系统的设计与开发,协助完成了核心功能。"
✅ 改为:"主导 Orchestrator 模块设计(代码贡献 70%),设计了 5 个核心 Agent 的通信协议。"

❌ **成果没有数字**
> "上线后效果显著,得到了用户的一致好评。"
✅ 改为:"上线 3 个月日均处理 6500 工单,自动解决率 73%,NPS 提升 18 分。"
