# JD 关键词映射与简历策略

针对不同公司类型,提供差异化的简历优化策略。

## 一、公司分类与定位

### 1. 一线大厂(字节跳动 / 阿里 / 腾讯 / 百度 / 美团 / 京东)

**典型岗位描述模式**:
- "负责 XX 业务线的 LLM Agent 系统设计与落地"
- "构建高并发、低延迟的智能对话系统"
- "完善 Agent 评估体系与可观测性"

**HR 筛简历关注**:
- 学历(普通本科以上,985/211 优先)
- 大厂背景(同级别公司互相认可)
- 项目规模(DAU、QPS、营收影响)
- 稳定性(单家公司 ≥ 1.5 年)

**技术面关注**:
- 系统设计能力(画架构图、讲扩展性)
- 工程化深度(评估、监控、降本)
- 解决疑难问题的经验

**简历策略**:
- ✅ 突出**规模数字**:DAU、调用量、token 消耗、成本节省
- ✅ 突出**工程化**:可观测性、评估体系、A/B Testing
- ✅ 突出**跨团队协作**:对接产品 / 运营 / 算法 / 基础架构
- ❌ 不要过度堆砌新名词,大厂更看重落地深度
- ❌ 不要写跳槽频繁(< 1 年的经历淡化处理)

### 2. AI 头部初创(MiniMax / 智谱 / 月之暗面 Kimi / DeepSeek / 阶跃星辰)

**典型岗位描述模式**:
- "推动 Agent 范式创新,探索前沿技术落地"
- "参与公司核心产品的 Agent 能力建设"
- "对接客户场景,快速验证 PMF"

**HR/创始人关注**:
- 技术热情(GitHub、博客、社区活跃度)
- 0-1 项目经验
- 学习能力与适应速度
- 对 AGI 的理解与品味

**简历策略**:
- ✅ 突出**开源贡献 / 个人项目**(给主流框架提 PR、高 star 项目)
- ✅ 突出**技术品味**(读过哪些论文、做过哪些前沿尝试)
- ✅ 突出**0-1 经验**(从无到有搭建过 Agent 系统)
- ✅ 写**技术博客 / 论文**链接(可放二维码或短链)
- ❌ 不要写得太"螺丝钉"(初创要的是能独立扛事的人)

### 3. 外企(Microsoft / Google / Anthropic / OpenAI 中国 / Databricks)

**典型岗位描述模式(英文)**:
- "Build production-grade LLM applications and agent systems"
- "Drive technical decisions and influence team direction"
- "Collaborate with global teams across time zones"

**HR 筛简历关注**:
- 英文能力(简历必须英文 + 沟通流畅)
- 教育背景(海外硕士/Top 校加分)
- OSS 贡献(GitHub 是硬通货)
- Ownership 体现

**简历策略**:
- ✅ **必须英文版** + 中文版备份
- ✅ 强调 **"I led / I designed / I shipped"**,不要 "we"
- ✅ 突出 **Impact**(用具体数字 + 业务上下文)
- ✅ 写 **OSS contributions**(commit 数、被合并的 PR、stars)
- ✅ 用 **STAR 法**严格组织,每段开头一句话总结
- ❌ 不要过度自谦("协助"、"参与")
- ❌ 不要写无关认证(雅思托福写在最后即可)

### 4. AI 应用公司(SaaS / 电商 / 教育 / 金融 / 法律)

**典型岗位描述模式**:
- "为 XX 行业客户构建定制化 Agent 解决方案"
- "理解客户业务场景,设计端到端 AI 产品"
- "推动 AI 能力的商业化落地"

**HR/业务关注**:
- 业务理解力
- 客户沟通能力
- 落地经验(签了多少客户、ROI)
- 行业经验(同行业优先)

**简历策略**:
- ✅ 突出**业务价值**:客户数、ARR、续约率、ROI
- ✅ 突出**行业 know-how**:对垂直领域的理解
- ✅ 突出**端到端能力**:从需求到上线全流程
- ❌ 不要过度炫技,业务方更关心结果

### 5. 传统行业数字化部门(银行 / 保险 / 制造 / 能源)

**典型岗位描述模式**:
- "推动公司智能化转型,落地 AI 助手"
- "对接业务部门需求,设计 AI 应用"
- "保障 AI 系统的合规与安全"

**HR 关注**:
- 学历(国企更看重)
- 稳定性(忌跳槽频繁)
- 合规意识(等保 / 隐私 / 信创)

**简历策略**:
- ✅ 突出**合规与安全**经验(数据脱敏、内容审核、私有化部署)
- ✅ 突出**国产化适配**(国产模型、信创环境)
- ✅ 突出**稳健落地**(完善的测试、灰度、回滚)
- ❌ 不要太追求新名词,传统行业更看重稳定可靠

## 三、不同年限的简历重心

| 年限 | 简历重心 | 字数控制 |
|------|---------|---------|
| < 1 年(应届 / 转型) | 项目深度 + 学习能力 + 个人项目 / 实习 | 1 页 |
| 1-3 年 | 2-3 个核心项目 + 量化成果 + 技术成长 | 1-1.5 页 |
| 3-5 年 | 主导项目 + 团队协作 + 技术影响力 | 1.5-2 页 |
| 5-8 年 | 架构能力 + 业务影响 + 团队管理(如有) | 2 页 |
| 8 年+ | 战略思考 + 业务结果 + 组织建设 | 2 页(简洁) |

## 四、薪资期望与岗位级别参考(2026,一线城市)

> ⚠️ 数据仅供简历期望栏参考,不代表实际市场行情

| 级别 | 大厂(腾讯T) | 字节(2-X) | 阿里(P) | AI 初创 |
|------|------------|----------|--------|---------|
| 初级(1-3 年) | T8-T9 / 30-50W | 2-1 ~ 2-2 / 35-60W | P5-P6 / 30-55W | 30-60W + 期权 |
| 中级(3-5 年) | T10-T11 / 50-100W | 3-1 ~ 3-2 / 60-120W | P6-P7 / 55-110W | 60-150W + 期权 |
| 高级(5-8 年) | T12 / 100-180W | 3-2 ~ 4-1 / 120-250W | P7-P8 / 110-250W | 150-300W + 期权 |
| 资深(8+ 年) | T13+ / 180W+ | 4-1+ / 250W+ | P8+ / 250W+ | 议价空间大 |

**简历中期望薪资写法**:
- 推荐:**"面议"** 或 **不写**,留出谈判空间
- 如必须写:写一个区间,**下限是当前薪资 × 1.3,上限是 × 1.6**
