# 简历自检清单

简历生成后,逐项检查。**任何一项不过关都要回去改**。

## 一、内容质量(最重要)

### ✅ 必查项
- [ ] 每个项目都有**至少 3 个量化指标**(数字、百分比、金额、时间)
- [ ] 每个项目都明确写出**"个人贡献"**(用 "主导"、"设计"、"实现"、"解决",不用"参与"、"协助")
- [ ] 技术栈不堆砌,**精通级别不超过 4 个**,且都在项目中体现
- [ ] 项目按**与目标 JD 的相关度排序**,最相关的在前
- [ ] 没有过期技术(如 2023 年前的 LangChain 0.0.x 单独列出来)

### ❌ 禁用词(出现一次扣 5 分)
- "负责" → 改为"主导 / 设计 / 实现 / 推动"
- "参与" → 改为"独立完成 X 模块 / 贡献 Y% 代码"
- "协助" → 删除,或改为具体动作
- "熟练掌握" + 一堆名词 → 改为分级表达 + 在哪个项目用过
- "提升了效率 / 优化了性能" → 必须给数字
- "采用先进技术" → 写出具体技术名
- "深入研究" → 改为"读过 X 源码 / 输出 Y 篇博客"
- "技术大牛 / 极客 / 强烈兴趣" → 全删,自吹无意义

### ⚠️ 警告词(谨慎使用,确保能扛住面试)
- "精通" → 必须能讲源码级原理
- "架构师" → 必须主导过 ≥3 人团队的设计
- "全栈" → 前后端必须都有上线项目
- "AI 专家 / Agent 专家" → 除非有论文 / 千 star 项目

## 二、结构与排版

- [ ] **一页打完**(中文 1 页,3-5 年经验最多 2 页,超过即冗余)
- [ ] 标题层级清晰(姓名 > 求职意向 > 技能 > 项目 > 工作 > 教育)
- [ ] **没有照片**(国内技术岗位除非要求,一律不放)
- [ ] **没有以下无关信息**:籍贯、政治面貌、婚姻状况、身高、星座
- [ ] 每个项目 5-8 行,过长压缩,过短补充
- [ ] 时间格式统一(推荐 `YYYY.MM - YYYY.MM` 或 `YYYY.MM - 至今`)

## 三、关键词命中(对照目标 JD)

把目标 JD 复制到一边,逐个 check:

- [ ] JD 中提到的**框架** (如 LangGraph、AutoGen) 是否在简历出现
- [ ] JD 中提到的**模型** (Claude、GPT、开源模型) 是否在简历出现
- [ ] JD 中提到的**场景** (RAG、多 Agent、Computer Use) 是否在项目体现
- [ ] JD 中的**编程语言** (Python、TypeScript) 是否标注熟练度
- [ ] JD 中的**软技能** (跨团队协作、产品 sense) 是否在工作经历体现

**目标:JD 关键词覆盖 ≥ 70%**

## 四、Agent 工程师专项检查

应聘 Agent 岗位特有的检查项:

- [ ] 至少有 **1 个端到端的 Agent 项目**(不只是套 ChatGPT API)
- [ ] 体现了 **Agent 工程化能力**(评估 / 监控 / 部署 / 降本)
- [ ] 体现了 **Prompt 工程深度**(不只是写了 prompt,而是测试 / 迭代 / 系统化管理)
- [ ] 体现了 **业务理解**(知道 Agent 解决什么业务问题,不是炫技)
- [ ] 跟进了 **2025-2026 新技术**(MCP / Computer Use / Agent SDK / 长上下文等任意 1-2 个)
- [ ] 如果是大厂岗:体现了 **规模化思考**(高并发、成本、稳定性)
- [ ] 如果是初创岗:体现了 **快速迭代能力**(0-1 项目经验、开源贡献)

## 五、真实性与可面试性

- [ ] 简历上每一句话,**面试时被追问都能讲 5 分钟**
- [ ] 数字都是真实的(面试官可能问"怎么算的")
- [ ] 技术选型能讲清楚 **为什么选 A 不选 B**
- [ ] 踩过的坑能讲出具体 case
- [ ] 项目时间线合理(不要 3 个月做出别人 1 年的成果)

## 六、加分项检查

下列任一项有就加上,没有不强求:

- [ ] **GitHub 链接** + 高质量 repo(≥ 100 stars 的项目)
- [ ] **个人博客 / 公众号** + 高质量技术文章
- [ ] **开源贡献**(给 LangChain / LlamaIndex / vLLM 等主流项目提过 PR)
- [ ] **论文**(EMNLP / ACL / NeurIPS / ICLR / KDD industry track)
- [ ] **社区分享**(QCon / ArchSummit / 公司技术大会主题演讲)
- [ ] **专利**(发明专利,实用新型一般不写)
- [ ] **竞赛**(Kaggle 金牌、天池 Top 10、ACM 区域赛金牌)

## 七、最终输出前

- [ ] 通读 3 遍,无错别字、无语病
- [ ] 用 Markdown 渲染一遍,排版无破损
- [ ] 转 PDF 后检查格式,中英文混排无错位
- [ ] 文件命名规范:`简历_张三_AgentEngineer_2026.pdf`
- [ ] 备份一份不同岗位定制版本(大厂版 / 初创版 / 外企版)
